新しいチャットでの引き継ぎ用。両ファイルの最新版と一緒に渡してください。

## 直近の変更履歴

- **食事休憩の手動終了ボタンを実装。** 本仕様書には以前から「食事休憩中は数字表示を隠し `⏭ 食事休憩を終了` ボタンを表示する」という仕様が記載されていたが、`lock_window.ahk` には未実装だった（`OnMealEnd` 関数・`mealEndBtn` コントロールともに存在しなかった）。今回追加し、仕様と実装を一致させた。自動終了（時間帯超過）・手動終了（ボタン）の両方が `EndMealPauseResume()` という共通関数を呼ぶ形に整理。
- **クロッキーと食事休憩の干渉バグを修正。** `CheckMealPause` の冒頭で `isCroquis` なら早期リターンするようにした（詳細は下記「既知の問題・注意点」参照）。
- **重複コードの削除。** `lock_window.ahk` で `workWindowTitles` / `workWindowProcesses` が2箇所に重複定義されていた（コメント見出しごと丸ごとコピペされたもの）ため、古い方を削除。
- **launcher.ahk のデッドコード削除。** 就寝ブロックの自動スケジュール方式（`ScheduleNightBlock` と、それが使うPS1事前生成処理）は手動トリガー方式への移行後は呼ばれておらず、起動のたびに無駄なファイル書き込みが発生していたため削除。あわせて、同じく使われていなかった `night_debug.txt` デバッグログと `nightBlockHourStart/End`（時間帯判定用・未使用）設定も削除。
- **NextDNS関連変数の命名統一。** launcher.ahk 側だけ `nightdnsApiKey` 等「night」接頭辞になっており、lock_window.ahk 側の `nextdnsApiKey` 等「next」接頭辞と紛らわしかったため、launcher.ahk 側も `nextdns*` に統一。
- **スリープ解除ルーティンの手動実行機能を追加。** トレイメニューに「🔁 スリープ解除ルーティンを手動実行」を追加。自動起床時の処理（就寝ブロック解除→待機→クロッキー→作業）を `StartWakeRoutine()` として共通化し、手動メニューからも同じ関数を呼べるようにした。二重起動防止のため `wakeRoutineActive` フラグと `current_phase.txt` のチェックを追加。

## launcher.ahk の機能

### スリープ復帰検知・起動フロー

1. スリープ時間が `minSleepHours`（6時間）以上かつ `skipWDays`（土曜）でなければ起動
2. 復帰から `delayMinutes`（30分）後にカウントダウン終了
3. 食事時間帯（19:00〜20:30）内であればクロッキーを食事休憩終了まで待機
4. クロッキーセッション開始（`lock_window.ahk /croquis:...` を起動）
5. クロッキー終了後（`current_phase.txt` が `croquis_done` になったことを2秒ごとに検知）、通常タイマー起動（`lock_window.ahk /auto`）

- スリープ前に `last_sleep.txt` に時刻を記録（30分以内の短時間スリープは上書きしない）
- 復帰後にカウントダウンGUIを表示（240×120px、右に「今すぐ開始」ボタン付き）
- トレイアイコンのツールチップにも残り時間を表示
- トレイアイコンのダブルクリックでGUIを最前面に戻せる

**手動実行（トレイメニュー「🔁 スリープ解除ルーティンを手動実行」）**

バグや寝落ちでスリープ復帰イベントが検知されなかった場合などのために、上記2〜5（待機→クロッキー→作業）を手動で開始できる。自動起床時と共通の `StartWakeRoutine()` を呼ぶため流れは完全に同じだが、`skipWDays` や `minSleepHours` によるスキップ判定は行わない（手動実行は常に開始する）。以下の場合は実行されない：
- ルーティンが既に進行中（`wakeRoutineActive` フラグ）
- `lock_window.ahk` が既に起動中（`current_phase.txt` が空でない）

いずれの場合もTrayTipで理由を通知するのみで、何も開始しない。

### クロッキー機能

- モード選択（`croquisMode`）: 1=25分×1枚 / 2=15分×2枚 / 3=5分×5枚
- 各モードのパラメータは `croquisModeParams` Map で定義（lockSecs/sets/interSecs）
- パラメータを `/croquis:lockSecs:sets:interSecs` の形式で lock_window.ahk に引数渡し
- モデル画像を `croquisFolder` からランダム選択し、GDI+でクリップボードにコピー（ビューアは開かない）
- 使用済み画像は `croquis_used.txt` に記録し全周でリセット
- 画像フォルダが存在しない・画像がない場合はクロッキーをスキップして通常タイマーへ

### クロッキー週次レポート

- 毎週土曜15時に `croquis_shots` フォルダの過去7日分の画像をDiscordに送信
- 送信先: `croquisReportWebhook`（サボり通知とは別のWebhook URL設定済み）
- テキスト（週・枚数・実施日一覧）＋画像をmultipart/form-dataで送信（最大10枚）
- PowerShellを `RunWait` で同期実行し、完了後にTrayTipを表示
- 日付の昇順ソートは `StrCompare` を使用（数値比較エラーを避けるため）

### サボり監視（CheckAbsence・30秒ごと）

- `current_phase.txt` が `lock` または `intermission` のときのみ監視
- 監視対象外フェーズ（lunch/exercise/break等）に入ったら sabo 状態を全リセット（isAbsent/warnedThisIdle/absentStartMs）
- フェーズが監視対象外→監視対象に**戻ったときも**同様にリセット（`sabo.lastPhase` で前回フェーズを保持）
  → これにより食事休憩中の無操作が復帰後にサボりとして誤記録されるのを防ぐ
- `absentThresholdMin`（15分）無操作でサボり判定
- 判定1分前にTrayTip通知＋`saboWarnWebhook`（専用Webhook）にDiscord送信
- 復帰時に`discordWebhook`（通常Webhook）にDiscord通知（離席時間を記録）
- 毎日23時に `sabo_logs/yyyy-MM-dd_sabo.txt` にログ書き出し
- 翌朝スリープ解除時に前日ログをGUI表示
- 異常終了（sentinel ファイルで次回起動時に検知）はログに記録

### Discord通知タイミング（launcher側）

- サボり警告（閾値1分前）→ `saboWarnWebhook` に送信
- サボり復帰確認（離席N分）→ `discordWebhook` に送信
- launcher.ahk 終了時（リロード時は通知しない）→ `discordWebhook` に送信

### 就寝スマホブロック

- トレイメニュー「🌙 就寝モード開始」→ 10秒後にモニターOFF、`nightBlockDelay`（30分）後にNextDNSでスマホをブロック＋PCをスリープ
- 翌朝スリープ解除時（6時間以上経過）に `night_block_flag.txt` を確認しNextDNSブロック解除
- 就寝ブロックは手動トリガーのみ（旧・自動スケジュール方式の関連コードは削除済み）

## lock_window.ahk の機能

### 起動モード

- **自動起動**（`/auto` 引数）：GUIなしで即スタート。siteList全件＋Steamをブロック
- **クロッキーモード**（`/croquis:lockSecs:sets:interSecs` 引数）：クロッキー専用タイマー。下記参照
- **手動起動**：GUIで Lock時間・Break時間・セット数を選択してStart

### クロッキーモード（/croquis）

- 引数形式: `/croquis:1500:1:0`（lockSecs:sets:interSecs）
- 集中モード強制ON・NextDNSブロック・GUIを紫色で表示
- セット数が複数の場合（モード2・3）:
  - 各セット終了 → 撮影猶予カウントダウン（`croquisCaptureWait`=10秒）→ キャプチャ → NextDNS解除 → セット間休憩 → 次の画像をクリップボードにコピー → 次セット開始
  - **注意**: `CaptureCountdownTick` はネスト関数のクロージャ問題を避けるためトップレベル関数として定義し、必要な変数は `croquisCaptureEnd` 等のグローバル変数で受け渡している
- 全セット終了後 → 最終休憩（`croquisBreakSecs`=300秒）→ `WritePhase("croquis_done")` → `Sleep(8000)` → `ExitApp()`
- launcher.ahk 側が2秒ごとに `current_phase.txt` を監視して `croquis_done` を検知し通常タイマーを起動
- `CleanupPhaseFile`（OnExit）: `current_phase.txt` の実際の内容（`actualPhase`）を読み、`croquis_done` のときはファイルを消去しない

### キャプチャ（CaptureCroquisResult）

- タイマー終了の撮影猶予中に自動実行（ユーザー操作なし）
- CLIPStudioPaintを前面に強制表示（0.8秒待機）してからウィンドウキャプチャ
- CLIPStudioが見つからない場合はプライマリモニター全体にフォールバック
- 保存先: `croquis_shots\yyyy-MM-dd.png`（同日2枚目以降は `yyyy-MM-dd_2.png` と連番）
- PowerShellを `RunWait` で同期実行

### 次セット間の画像選択（lock_window.ahk 側）

- `PickNextCroquisImage()` / `CopyNextCroquisImage()` がlauncher側と同じロジックで画像を選択
- 画像フォルダは `A_ScriptDir\croquis_models`（ハードコード）
- 使用済みリストは `croquis_used.txt`（launcher側と共有）

### ポモドーロタイマー（/auto・手動）

- ロック → 休憩 → ロック … を指定セット数繰り返す
- セット完了後は「中休みモード」（15分）に移行
- 中休みで「作業完了」ボタンを押すと本当のdoneモードへ（**作業達成済みのときのみ表示**）
- 中休みタイムアウト → 2セット追加して再開
- ➕ボタンでセット追加（中休み・done中でも再開可能）
- `RunPomodoro` は generation カウンターでタイマー競合を防止

### ウィンドウブロック

- ロック中、`siteList` に含まれるタイトルのウィンドウを300msごとに最小化
- ロック開始/終了時に NextDNS でスマホのSNS・動画・ゲーム系もブロック/解除

### 一時停止系モード（タイマー・サボり検知・ブロックが停止）

| モード | トリガー | 時間 |
|---|---|---|
| 🍽️ 食事休憩 | 毎日19:00〜20:30 自動 | 固定 |
| 🚴 運動モード | ボタン（1日1回） | 60分 |
| 🥗 昼休みモード | ボタン（1日1回） | 50分 |

- これらは `g.isExercise` フラグを共用し `ResumeAfterExercise()` で復帰
- 食事休憩開始時: `WritePhase("lunch")` でサボり検知を無効化、`timerCount.Visible := false` で数字表示を隠し代わりに「⏭ 食事休憩を終了」ボタン（y=38, w=160, h=36）を表示
- 食事休憩の手動終了ボタン（`OnMealEnd`）: 押すと通常の食事休憩終了と同じ処理を実行し `timerCount.Visible := true` に戻す
- 食事休憩終了時: `WritePhase(g.phase)` で元のフェーズに戻し、NextDNSブロックを復帰
- `ResumeAfterExercise()` の食事休憩中分岐でも `NextDnsUnblock()` と `timerCount.Visible := true` を実行
- 一時停止中に `g.isPaused = true` になると `g.lastActiveCheck` が更新されないため、復帰時に `g.lastActiveCheck := 0` をリセット（食事/運動どちらも）→ 作業時間への誤加算防止

### 集中モード（🎯）

- 許可リスト（`focusModeAllowList`/`focusModeAllowProcesses`）以外の全ウィンドウを最小化
- **自動発動**：`focusModeAutoFromSet`（3）セット目以降、`focusModeAutoChance`（50%）の確率でON。そのセットのみ有効（auto フラグで管理）
- **手動発動**：ロック中にボタンON → 休憩中のみOFF可能 → OFFしなければ次セットへ引き継ぎ
- 発動時 `focusModeCountdownSecs`（15秒）の猶予あり（猶予中は通常ブロックも一時停止）
- 終了時に最小化したウィンドウを復元（`focusMinimizedHwnds` に記録）

### 休憩延期

- ロック終了時に `workWindowProcesses`（CLIPStudio等）がアクティブなら休憩を延期
- 偽装タイマー（`breakDeferFakeMinutes`=3分）を表示（0になったらリセット繰り返し）
- 作業ウィンドウ以外が `breakDeferGraceSecs`（5秒）以上続いたら休憩開始

### 作業達成時間

- `totalWorkGoalMinutes`（150分=2時間半）の間、`workWindowProcesses` がアクティブになると達成
- **クロッキー中も計測される**（`CheckActiveWork` タイマーはスクリプト起動時から常時動作し、`g.phase = "lock"` かつ `g.isPaused = false` なら計測）
- 達成時：TrayTip通知（Discord通知は送らない）
- 達成前：中休み中の「✅ 完了」ボタンは `workDoneBtn.Visible := false`（非表示）
- 達成後に中休みに入った場合は即座に表示。中休み中に達成した場合も即座に表示

### ゲームプレイ時間制限

- `gameLimitMinutes`（120分）を超えると `WinClose` で通常終了＋以降最小化し続ける
- 制限後、作業セットを `gameLimitExtendSets`（1セット）完了すると1日1回延長可能
- プレイ時間は `game_limit.txt` に保存、日付が変わるとリセット

### フェーズファイル（current_phase.txt）

launcher.ahk との通信手段。

| 値 | 意味 |
|---|---|
| `lock` | サボり監視ON |
| `intermission` | サボり監視ON |
| `break` / `done` / `lunch` / `exercise` | サボり監視OFF |
| `croquis_done` | クロッキー正常終了（launcherが読んだ後に削除） |

### Discord通知タイミング（lock_window側）

- ロック中に終了 → 通知（クロッキー中 `isCroquis=true` の場合は送らない）
- リロード時は通知しない
- 作業達成・ゲーム制限到達時は通知しない（TrayTipのみ）

## 既知の問題・注意点

### クロッキーと食事休憩の干渉（修正済み）

クロッキーのロックタイマー終了直前（残り約300ms以下）のタイミングで `CheckMealPause`（15秒ごと）が発火すると、`g.isPaused = true` がセットされ `CroquisLockTick` が `remaining <= 0` の分岐に到達できなくなる問題があった。**修正済み：** `CheckMealPause` の冒頭で `isCroquis` なら即 return するようにした。これによりクロッキー中は食事休憩による中断を一切受けず、最後まで進行してから通常タイマーに戻る（launcher 側が食事時間帯の開始判定を行っているため、食事時間帯に食い込むのは稀なケース）。

### CaptureCountdownTick のトップレベル化

AHKv2 の多重ネスト関数ではクロージャが正しく動作しない場合があるため、`CaptureCountdownTick` をトップレベル関数として定義し、必要な値（`captureEnd` 等）を `croquisCaptureEnd` 等のグローバル変数で受け渡している。

### ゲームプレイ制限の「1日1回延長」機能（未実装・対応保留）

`gameLimitExtendSets` や `g.gameExtendSetsLeft` / `g.gameExtendUsed` / `gameExtendSetsDone` は変数として用意されているだけで、実際に延長を判定・付与する処理が存在しない。`g.gameLimitReached` を `false` に戻す経路がどこにもないため、**一度制限に達するとそのセッション中は解除されない。** 実装するなら、中休み（intermission）やセット完了のタイミングで完了セット数をカウントし、`gameLimitExtendSets` に達したら `g.gameLimitReached := false` にリセットする処理を追加する必要がある。

### settings_editor.html

- ブラウザで開いてAHKファイルをドロップ → 値を編集 → 「保存」でダウンロード → 元ファイルに上書き
- 保存時に自動でバックアップ（`.ahk.backup`）を作成
- ファイル内の変数を正規表現でパースするため、変数名が他の文字列中に含まれるケースでは誤マッチする可能性がある（現状の設定変数では問題なし）
